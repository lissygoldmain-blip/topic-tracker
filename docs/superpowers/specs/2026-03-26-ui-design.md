# Topic Tracker UI — Design Spec
**Date:** 2026-03-26
**Status:** Approved

---

## Overview

A lightweight personal dashboard for reading Topic Tracker results. Hosted on GitHub Pages (`topic-tracker-ui` repo). Works on phone and laptop. No build step, no framework, no server.

Primary use: quick daily triage ("did anything important happen?"). Secondary: browsing results by topic when there's time.

---

## Architecture

**Tech stack:** Vanilla HTML, CSS, JavaScript — three files, no dependencies, no build pipeline.

**Files:**
```
index.html   — app shell, tab bar, screen containers
style.css    — all styles (CSS custom properties for theming)
app.js       — all logic (ES module, fetch, DOM manipulation)
```

**Page title:** `<title>Topic Tracker</title>` with a favicon to match.

**Data source:** `results/index.json` — pushed to the `topic-tracker-ui` repo by `poll.yml` after each poller run. Topic names come from the keys of this file; no `topics.yaml` is fetched. Shape:
```json
{
  "Topic Name": [
    {
      "url": "...",
      "title": "...",
      "snippet": "...",
      "source": "google_news",
      "source_type": "news",
      "topic_name": "Topic Name",
      "fetched_at": "2026-03-26T10:00:00+00:00",
      "novelty_score": 0.87,
      "summary": "...",
      "tags": ["new_listing"],
      "escalation_trigger": null,
      "price": null,
      "notified_push": false,
      "notified_digest": true
    }
  ]
}
```

**State persistence:** Read/unread status tracked in `localStorage` as a JSON array (reconstructed as a `Set` in memory). Key: `tracker_read_urls`. Storage format: `JSON.stringify([...readSet])`. On load: `new Set(JSON.parse(localStorage.getItem('tracker_read_urls') || '[]'))`.

**Hosting:** GitHub Pages on the existing `topic-tracker-ui` repo. Accessible at `https://{owner}.github.io/topic-tracker-ui`.

---

## Navigation

Bottom tab bar with three tabs, persistent across all screens. Mobile-first; desktop uses the same layout initially (sidebar/top nav layout deferred to a future iteration).

```
[ Highlights ]  [ Topics ]  [ Settings ]
```

Active tab indicated by color/weight on the tab label/icon.

---

## Screen 1: Highlights

**Purpose:** "Did anything important happen?" — the default landing screen.

**What appears here:** Results where `novelty_score >= 0.8` OR `escalation_trigger` is not null. Escalation-triggered results appear regardless of their score.

**Sort order:** Unread results first (sorted by `novelty_score` descending within the unread group), then read results (sorted by `novelty_score` descending within the read group). This applies to both Highlights and Topics results views consistently.

**Empty state:** A calm message: "Nothing new to highlight." — not an error, just a quiet confirmation.

**Result card** (each result in the list):
- Topic name as a small colored pill (top left)
- Title (primary text, 1–2 lines)
- Snippet (secondary text, 2 lines, truncated with ellipsis)
- Source name + relative time — small, bottom left. Formatting rules:
  - < 1 hour: "Xm ago"
  - 1–48 hours: "Xh ago"
  - Older: "MMM D" (e.g. "Mar 24")
- Novelty indicator — a small colored dot. Thresholds: score ≥ 0.9 = green; 0.8–0.89 = yellow. Shown only on Highlights (all cards qualify since threshold is ≥ 0.8). Not shown on Topics results view to avoid visual noise on lower-scored items.
- Escalation badge — if `escalation_trigger` is set, a small inline badge showing the trigger tag (e.g. "⚡ drop_confirmed")
- Read results shown at reduced opacity (0.5)

**Detail sheet** (slides up from bottom on tap, full-width modal on desktop):
- Visible close button (×) in top-right corner
- Also dismissible by: tapping/clicking outside the sheet, pressing Escape
- Title (large)
- Source + published time (absolute, e.g. "Mar 24, 2026 10:42")
- Summary (full text if available, otherwise snippet)
- Tags as pills
- Novelty score (numeric, e.g. "0.87")
- Price (if set — shopping results)
- Escalation trigger (if set, prominent amber badge)
- "Open →" button — opens `url` in a new tab
- Opening the detail sheet marks the result as read (adds URL to `tracker_read_urls`)

## Screen 2: Topics

**Purpose:** Browse all topics and dive into any one's results.

**Topic list view:**
- One row per topic, derived from the keys of `index.json`
- Shows: topic name, total result count
- Escalation indicator: a small ⚡ badge if any result for that topic has a non-null `escalation_trigger`. Derived entirely from `index.json` — no additional file fetch needed.
- Topic description not shown (keeps the list clean)
- Tap a row → topic results view

**Topic results view:**
- Back button → returns to topic list
- Topic name as heading
- All results for that topic, sorted by the same rule as Highlights: unread first (by novelty desc), then read (by novelty desc)
- Same card format as Highlights, but without the topic pill (topic is already the heading) and without the novelty dot
- Same detail sheet on tap
- **Empty state:** "No results yet for this topic." — shown if the topic key exists in `index.json` but has an empty array

---

## Screen 3: Settings

**Purpose:** Utility screen. Minimal in v1.

**Contents:**
- **Refresh** button — re-fetches `index.json` without a full page reload. During fetch: button disabled, label changes to "Refreshing…". On success: brief "Updated" confirmation (2s), then reverts. On error: "Couldn't refresh — try again."
- **Last fetched** — the maximum `fetched_at` value across all results in `index.json` (i.e. when the poller last ran). Computed by comparing `new Date(fetched_at).getTime()` values (not string comparison, which breaks with mixed UTC offsets). Formatted as an absolute datetime. Fallback: if `index.json` has no results yet, show "—".
- **GitHub token** — omitted from v1 UI entirely. Will be added in the topic creation feature.

## Visual Design

**Aesthetic:** Clean, minimal, information-dense. Inspired by linear.app / Notion aesthetic. No heavy UI library.

**Theming:** CSS custom properties for all colors. Follows system `prefers-color-scheme` automatically (light mode default, dark mode when OS is dark).

**Typography:** System font stack (`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`). No external font loading.

**Topic pills:** Each topic gets a consistent color derived from its name (simple hash → hue). Subtle background, colored text.

**Spacing:** Generous padding on cards. 16px base unit.

**Escalation badge:** Amber/yellow color, small, inline. Uses ⚡ for quick visual recognition.

## Data Loading

1. On app load: `fetch('results/index.json')`
2. Parse response into a map of topic → results and a flat array of all results
3. Compute highlights: results where `novelty_score >= 0.8` OR `escalation_trigger !== null`
4. Load read set from `localStorage`
5. Render active tab
6. On refresh: repeat steps 1–5, preserve read set

**Error handling:** Network errors and non-404 HTTP failures show a friendly error state with a retry button. A 404 (poller hasn't run yet) is treated as empty data — show the normal UI with empty states ("Nothing new to highlight.", topic list shows no rows) rather than an error. No crash in either case.

---

## Read/Unread State

- Stored in `localStorage` under key `tracker_read_urls` as a JSON array of URLs
- Reconstructed as a `Set` in memory on load
- Opening a detail sheet adds the result's URL to the set and immediately saves back to `localStorage`
- Read cards shown at reduced opacity (0.5)
- Sort order everywhere: unread (by novelty desc) before read (by novelty desc)
- No "mark all read" in v1 — too easy to accidentally clear things

## Out of Scope (v1)

- Topic creation / editing (future — needs GitHub API + LLM-assisted wizard)
- Desktop-specific layout (sidebar navigation)
- Push notifications
- Search / full-text filtering
- Pagination (load all results; revisit if performance is an issue)
- Authentication / multi-user
- GitHub token / Settings placeholders

## Future Considerations

- **Topic creation wizard**: LLM-assisted form that takes a short description and generates a full `TopicConfig` (terms, sources, thresholds, escalation triggers), commits to the tracker repo via GitHub API using a stored PAT.
- **Desktop layout**: Sidebar navigation replacing the tab bar when viewport ≥ 768px.
- **Richer visual design**: Topic-specific color themes, card art, more personality — once the functional foundation is solid.
