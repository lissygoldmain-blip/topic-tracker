# Topic Tracker UI — Design Spec
**Date:** 2026-03-26
**Status:** Approved

---

## Overview

A lightweight personal dashboard for reading Topic Tracker results. Hosted on GitHub Pages (`topic-tracker-ui` repo). Works on phone and laptop. No build step, no framework, no server.

Primary use: quick daily triage ("did anything important happen?"). Secondary: browsing results by topic when there's time.

---

## Architecture

**Tech stack:** Vanilla HTML, CSS, JavaScript — three files, no dependencies, no build pipeline.

**Files:**
```
index.html   — app shell, tab bar, screen containers
style.css    — all styles (CSS custom properties for theming)
app.js       — all logic (ES module, fetch, DOM manipulation)
```

**Data source:** `results/index.json` — already pushed to the `topic-tracker-ui` repo by `poll.yml` after each poller run. Shape:
```json
{
  "Topic Name": [
    {
      "url": "...",
      "title": "...",
      "snippet": "...",
      "source": "google_news",
      "source_type": "news",
      "topic_name": "Topic Name",
      "fetched_at": "2026-03-26T10:00:00+00:00",
      "novelty_score": 0.87,
      "summary": "...",
      "tags": ["new_listing"],
      "escalation_trigger": null,
      "price": null,
      "notified_push": false,
      "notified_digest": true
    }
  ]
}
```

**State persistence:** Read/unread status tracked in `localStorage` keyed by result URL. No backend required.

**Hosting:** GitHub Pages on the existing `topic-tracker-ui` repo. Accessible at `https://{owner}.github.io/topic-tracker-ui`.

---

## Navigation

Bottom tab bar with three tabs, persistent across all screens. Mobile-first; desktop uses the same layout initially (sidebar/top nav layout deferred to a future iteration).

```
[ Highlights ]  [ Topics ]  [ Settings ]
```

Active tab indicated by color/weight on the tab label/icon.

---

## Screen 1: Highlights

**Purpose:** "Did anything important happen?" — the default landing screen.

**What appears here:** Results where `novelty_score >= 0.8` OR `escalation_trigger` is not null. Sorted by `novelty_score` descending (escalated results float naturally since they tend to score high; if not, escalation badge makes them visually distinct).

**Empty state:** A calm message: "Nothing new to highlight." — not an error, just a quiet confirmation.

**Result card** (each result in the list):
- Topic name as a small colored pill (top left)
- Title (primary text, 1–2 lines)
- Snippet (secondary text, 2 lines, truncated with ellipsis)
- Source name + relative time ("6h ago", "2d ago") — small, bottom left
- Novelty indicator — a small colored dot (green = high, yellow = medium; only shown for scores ≥ 0.8 so all highlights cards show it)
- Escalation badge — if `escalation_trigger` is set, a small inline badge showing the trigger tag (e.g. "⚡ drop_confirmed")

**Detail sheet** (slides up from bottom on tap):
- Title (large)
- Source + published time
- Summary (full text if available, otherwise snippet)
- Tags as pills
- Novelty score (numeric, e.g. "0.87")
- Price (if set — shopping results)
- Escalation trigger (if set, prominent badge)
- "Open →" button — opens `url` in a new tab
- Dismiss by tapping outside or a close gesture

---

## Screen 2: Topics

**Purpose:** Browse all topics and dive into any one's results.

**Topic list view:**
- One row per topic from `topics.yaml` (derived from keys in `index.json`)
- Shows: topic name, total result count, escalation indicator (dot or badge if an escalation is active — read from `state.json` if available, otherwise omitted)
- Topic description is not shown by default (keeps the list clean)
- Tap a row → topic results view

**Topic results view:**
- Back button → returns to topic list
- Topic name as heading
- All results for that topic, sorted by `novelty_score` descending
- Same card format as Highlights (without the topic pill, since topic is already the heading)
- Same detail sheet on tap

---

## Screen 3: Settings

**Purpose:** Utility screen. Minimal in v1.

**Contents:**
- **Refresh** button — re-fetches `index.json` without a full page reload; updates the displayed data. Shows a brief "Updated" confirmation.
- **Last fetched** timestamp — when `index.json` was last loaded (from the most recent `fetched_at` value across all results, or the HTTP response date).
- **GitHub token** field (placeholder, non-functional in v1) — for future topic creation feature. Shows a note explaining it's coming soon.

## Visual Design

**Aesthetic:** Clean, minimal, information-dense. Inspired by linear.app / Notion aesthetic. No heavy UI library.

**Theming:** CSS custom properties for all colors. Follows system `prefers-color-scheme` automatically (light mode default, dark mode when OS is dark).

**Typography:** System font stack (`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`). No external font loading.

**Topic pills:** Each topic gets a consistent color derived from its name (simple hash → hue). Subtle background, colored text.

**Spacing:** Generous padding on cards. 16px base unit.

**Escalation badge:** Amber/yellow color, small, inline. Uses a lightning bolt character (⚡) for quick visual recognition.

---

## Data Loading

1. On app load: `fetch('results/index.json')`
2. Parse response into a flat array of all results + a map of topic → results
3. Compute highlights (novelty ≥ 0.8 or escalation_trigger set)
4. Render active tab
5. On refresh: repeat steps 1–4, preserve read/unread state from `localStorage`

**Error handling:** If fetch fails (offline, repo not yet populated), show a friendly error state with a retry button. No crash.

---

## Read/Unread State

- Tracked in `localStorage` as a Set of URLs: `tracker_read_urls`
- When a detail sheet is opened, the result's URL is added to the set
- Read results shown with reduced opacity (0.5) on their card
- On Highlights: read results move to the bottom (unread first)
- No "mark all read" in v1 — too easy to accidentally clear things

---

## Out of Scope (v1)

- Topic creation / editing (future — needs GitHub API + LLM-assisted wizard)
- Desktop-specific layout (sidebar navigation)
- Push notifications
- Search / full-text filtering
- Pagination (load all results; if perf is an issue, revisit)
- Authentication / multi-user

---

## Future Considerations

- **Topic creation wizard**: LLM-assisted form that takes a short description and generates a full `TopicConfig` (terms, sources, thresholds, escalation triggers), commits to the tracker repo via GitHub API using a stored PAT.
- **Desktop layout**: Sidebar navigation replacing the tab bar when viewport ≥ 768px.
- **Richer visual design**: Topic-specific color themes, card art, more personality — once the functional foundation is solid.
